# Phase 2.1: App-Context-Aware Prompts

> Date: 2026-05-03
> Status: Design

---

## Overview

Detect the frontmost macOS application when a dictation completes, and append a context hint to the AI system prompt so the LLM adjusts its grammar correction style to match the target app. For example, dictation pasted into Slack gets casual treatment while dictation pasted into Mail gets email-aware formatting.

Detection uses `NSWorkspace.shared.frontmostApplication.localizedName` — no Accessibility permission required beyond what iSpeak already has for paste simulation.

---

## Detection

**New file:** `src-tauri/src/frontmost_app.rs`

One public function:

```rust
pub fn get_frontmost_app_name() -> Option<String>
```

- Calls `NSWorkspace.shared.frontmostApplication.localizedName` via objc2 bindings
- Returns `Some("Slack")`, `Some("Mail")`, etc.
- Returns `None` on any failure (non-fatal)
- macOS only: `#[cfg(target_os = "macos")]`. On other platforms, returns `None`.

---

## Prompt Construction

**In `ai.rs`:**

The existing `SYSTEM_PROMPT` constant remains the base prompt (unchanged):

```
"You are a dictation assistant. The user spoke the following text aloud and it was
transcribed automatically. Fix grammar, punctuation, and capitalization errors. Do not
change meaning, tone, or wording beyond corrections. Output only the corrected text,
nothing else."
```

A new function `build_system_prompt(app_name: Option<&str>) -> String` appends a one-line context hint based on app category. If no app is detected or the app doesn't match any category, the base prompt is returned unmodified.

Format of the appended hint:

```
\nContext: The user is typing in {app_category}. {behaviour_hint}
```

---

## App Category Mapping

Match is case-insensitive against the app's `localizedName`. Uses `contains` matching so "Visual Studio Code" matches "Code".

| Category | Match strings | Hint |
|----------|--------------|------|
| Chat | `Slack`, `Teams`, `Discord` | Keep the tone casual, skip formal punctuation, preserve emoji intent. |
| Email | `Mail`, `Outlook`, `Gmail` | Use email formatting, be aware of greeting and sign-off structure. |
| Code editor | `VS Code`, `Code`, `Cursor`, `Zed` | Use minimal punctuation, be code-comment friendly, do not auto-capitalise. |
| Notes | `Notion`, `Obsidian` | Write clean prose, preserve bullet intent. |
| Terminal | `Terminal`, `iTerm`, `Warp`, `Ghostty` | Be command-friendly, do not capitalise, no trailing period. |
| Claude | `Claude` | Write clear prose, structured for LLM prompting. |
| Default | everything else | No hint appended. |

Match order matters for ambiguity: "Code" must be checked after "VS Code" to avoid matching unrelated apps. The match function checks the most specific strings first.

The full system prompt with hint must stay under 200 tokens (per SPEC.md). The base prompt is ~55 tokens; the longest hint adds ~25 tokens. Well within budget.

---

## Integration

In `ai.rs` `post_process()`:

1. Call `frontmost_app::get_frontmost_app_name()` to get the app name
2. Call `build_system_prompt(app_name.as_deref())` to construct the prompt
3. Pass the constructed prompt to `ollama_complete()` / `groq_chat_complete()` instead of the static `SYSTEM_PROMPT`

This requires changing `ollama_complete` and `groq_chat_complete` to accept the system prompt as a parameter instead of using the constant directly.

---

## Error Handling

- `get_frontmost_app_name()` failure → `None` → default prompt (no hint). Non-fatal, no error event emitted.
- Detection runs synchronously on the main thread (it's a single Objective-C call, effectively instant).
